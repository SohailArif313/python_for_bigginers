## 1Ô∏è‚É£ Functions in Python
- **Definition:** A function is a **block of code** designed to perform a specific task.  
- Functions help in making code **organized, reusable, and easier to maintain**.

### 1.1 Parameters and Arguments
- **Parameters:** Variables defined in the function to accept input.  
- **Arguments:** Actual values passed to the function when calling it.  
- **Example Uses in Day 6:**
  - Adding two numbers.  
  - Calculating the average of marks in multiple subjects.  

### 1.2 Return Values
- Functions can **return a value** using the `return` statement.  
- This allows the result to be **stored or used in further calculations**.  

### 1.3 Default Parameters
- Parameters can have **default values**, which are used if the user does not provide an argument.  
- Example: Multiplying numbers where one parameter has a default value.  

### 1.4 Calling Functions
- Functions are **executed by calling them** with required arguments.  
- Functions can be called **multiple times with different inputs**, promoting reusability.

## 2Ô∏è‚É£ Recursion
- **Definition:** Recursion is a technique where a function **calls itself** to solve a problem.  
- **Key Concepts:**
  - **Base Case:** The condition where recursion stops. Prevents infinite loops.  
  - **Recursive Case:** The part where the function calls itself with a smaller or simpler input.  

- **Uses in Day 6:**
  - Printing numbers from n to 1.  
  - Calculating the factorial of a number.  

- **Important Points:**
  - Always define a **base case**.  
  - Recursion can replace loops for **certain problems**, especially mathematical sequences.  
  - Prints or operations before the recursive call execute in **forward order**, and those after execute in **reverse order**.

## üí° Key Takeaways
- Functions make code **modular, readable, and reusable**.  
- Parameters, arguments, and return values make functions **flexible and dynamic**.  
- Default parameters provide **optional arguments**, simplifying function calls.  
- Recursion allows solving problems **by breaking them into smaller sub-problems**.  
- Always define a **base case** in recursion to prevent infinite calls.  
- Combining functions with recursion is powerful for **mathematical operations and structured programming**.
